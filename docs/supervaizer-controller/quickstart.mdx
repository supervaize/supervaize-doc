## <i className="bi bi-1-circle"></i> Clone the Hello World Example {#1}

:::info
The Supervaizer Controller currently supports **Python-based** agents.
:::

We'll use the **Supervaize Hello World** example ‚Äî a minimal agent that demonstrates all core Supervaizer concepts:
[<i className="bi bi-github"></i> GitHub Repository](https://github.com/supervaize/supervaize_hello_world)

```bash title="Clone the repo"
git clone https://github.com/supervaize/supervaize_hello_world.git
cd supervaize_hello_world
```

### Understanding the Controller

The Hello World example includes a pre-configured `supervaizer_control.py` that demonstrates:

- **Agent parameters** ‚Äî secrets and environment variables
- **Agent methods** ‚Äî start, stop, and status handlers
- **Server configuration** ‚Äî A2A protocol support and admin interface

```js reference title="Hello World controller supervaizer_control.py"
https://github.com/supervaize/supervaize_hello_world/blob/main/supervaizer_control.py#L1-L20
```

### Creating Your Own Controller

To scaffold a new controller for your own agent:

```bash title="Create template controller file"
supervaizer scaffold
# Success: Created an example file at supervaizer_control_example.py
```

For detailed instructions on customizing your controller, see the **[Controller Setup Guide](controller-setup)**.

## <i className="bi bi-3-circle"></i> Configure your Supervaize account & environment {#3}

### Supervaize account and credentials

[Create your developer account](/docs/supervaize-fleet/developer-account#getting_started) on the Supervaize platform.

Create your API Key and collect your environment variables.

:::tip
Use [direnv](https://direnv.net/) for easy management of environment variables in `.envrc` file.

Or setup [python-dotenv](https://github.com/theskumar/python-dotenv) in your python project.
:::

<Tabs groupId="pm" queryString>
  <TabItem value=".envrc" label=".envrc (direnv)">

```bash title="Export Environment variables in for .envrc"
export SUPERVAIZE_API_KEY=...
export SUPERVAIZE_WORKSPACE_ID=team_1
export SUPERVAIZE_API_URL=https://app.supervaize.com
```

    </TabItem>
      <TabItem value=".env" label=".env (dotenv)">


    </TabItem>

### For local development only {#localdev}

:::warning Local development
If the agent is not deployed to a publicly accessible server, you need to install a **Tunneling** solution so that `supervaize` can operate your agent.
Here are the instructions to [setup a tunnel for your local development](tunneling-setup)
:::

Store your public hostname in `SUPERVAIZER_HOST` environment variable.

```bash
export SUPERVAIZER_HOST=silly-peaches-chapter.loca.lt
```
